Controller: cron

This is an API endpoint that actually should be part of the API. These API requests are triggered by an admin user on the client and are responsible for progressing the game.

However, this a compromise due to the USBWebServer method of deployment. In practice, these commands should be run automatically on a timer by the server. In a live environment, this is relatively simple. In a portable one, it is not reasonably possible.

Default Method: None

Methods

executeturn

Execute a game turn, responsible mainly for generating resources

Arguments
    None
Responses
    None

executeCombatTurn

Execute a combat turn, responsible for progressing/resolving fleet combat

Arguments
    None
Responses
    None

executeMaintainence

Executes functions needing to be run at a very high frequency in order to simulate real time activity, such as fleet arrivals.

Arguments
    None
Responses
    None

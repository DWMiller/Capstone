Below is the only example of the actual format to be used in API requests
All other pages will list request formats in simple, nested lists with fields labelled as controller/method/arguments
Select controllers on the menu to the right to view details on regarding those controllers.

All request objects must be wrapped in an api object, as demonstrated above.
A controller is always required.
A method is required unless a default for the chosen controller provided.
When relying on a default method, arguments are not supported.
Arguments are optional, subject the requirements of each individual method.
